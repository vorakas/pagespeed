# PageSpeed Monitor — Design System

Generated by UI UX Pro Max | Based on existing dark theme audit

---

## Foundation

**Style:** Dark Mode (OLED) — WCAG AAA target
**Typography:** Inter (Minimal Swiss pairing — single font family, weight variations)
**Framework:** Vanilla HTML/CSS/JS with CSS Custom Properties

## 2. Typography Scale

**Font:** Inter (existing) — "Minimal Swiss" pairing from UI Pro Max database
**Rationale:** Best for dashboards, admin panels, and enterprise apps. Single family with weight variations for ultimate simplicity.

```css
:root {
  /* === Font Family === */
  --font-sans:  'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, Oxygen, Ubuntu, Cantarell, sans-serif;
  --font-mono:  'Cascadia Code', 'Fira Code', 'Consolas', 'Courier New', monospace;

  /* === Type Scale (1.25 Major Third) === */
  --text-xs:    0.75rem;   /* 12px — micro labels, badges */
  --text-sm:    0.875rem;  /* 14px — secondary text, table cells */
  --text-base:  1rem;      /* 16px — body text, form inputs */
  --text-lg:    1.125rem;  /* 18px — section subheadings */
  --text-xl:    1.25rem;   /* 20px — card titles, trigger names */
  --text-2xl:   1.5rem;    /* 24px — page section headings */
  --text-3xl:   2rem;      /* 32px — page title (h1) */
  --text-4xl:   2.5rem;    /* 40px — hero metrics, large numbers */

  /* === Font Weights === */
  --font-light:    300;
  --font-normal:   400;
  --font-medium:   500;
  --font-semibold: 600;
  --font-bold:     700;

  /* === Line Heights === */
  --leading-tight:  1.25;  /* Headings */
  --leading-normal: 1.5;   /* Body text */
  --leading-relaxed: 1.625; /* Long-form content */

  /* === Letter Spacing === */
  --tracking-tight:  -0.025em;  /* Large headings */
  --tracking-normal:  0;        /* Body text */
  --tracking-wide:    0.05em;   /* Uppercase labels, badges */
}
```

### Usage Guide

| Element | Size | Weight | Tracking |
|---------|------|--------|----------|
| Page title (h1) | `--text-3xl` | `--font-bold` | `--tracking-tight` |
| Section heading | `--text-2xl` | `--font-semibold` | `--tracking-tight` |
| Card title | `--text-xl` | `--font-semibold` | `--tracking-normal` |
| Body text | `--text-base` | `--font-normal` | `--tracking-normal` |
| Table cells | `--text-sm` | `--font-normal` | `--tracking-normal` |
| Badges / labels | `--text-xs` | `--font-medium` | `--tracking-wide` |
| Metric values | `--text-4xl` | `--font-bold` | `--tracking-tight` |
| Code / monospace | `--text-sm` | `--font-normal` | `--tracking-normal` |

## 3. Spacing Scale

Based on a **4px base unit** (harmonizes with the most common values already in use).

```css
:root {
  --space-1:   4px;    /* Micro — icon gaps, badge padding */
  --space-2:   8px;    /* XS — compact form elements, small gaps */
  --space-3:   12px;   /* SM — form inputs, table cells, nav links */
  --space-4:   16px;   /* MD — card content padding, standard gap */
  --space-5:   20px;   /* LG — section padding, nav padding */
  --space-6:   24px;   /* XL — major card padding */
  --space-8:   32px;   /* 2XL — page section headers */
  --space-10:  40px;   /* 3XL — large section spacing */
  --space-12:  48px;   /* 4XL — empty state padding, max spacing */
}
```

## 9. Accessibility Checklist

From UI Pro Max pre-delivery checklist + current audit findings:

- [x] No emojis as functional icons — using inline SVGs with `stroke: currentColor`
- [ ] `cursor: pointer` on all clickable elements (audit needed)
- [x] Hover states with transitions (200ms standard)
- [ ] Focus states visible for keyboard navigation (`:focus-visible` rings needed)
- [ ] `prefers-reduced-motion` respected (not yet implemented)
- [x] Text contrast: `--text-primary` on `--bg-base` = 15.4:1 (AAA)
- [x] Text contrast: `--text-secondary` on `--bg-base` = 12.8:1 (AAA)
- [ ] Text contrast: `--text-muted` on `--bg-base` = 3.7:1 (below AA 4.5:1 — use only for decorative text)
- [ ] Responsive breakpoints: 375px, 768px, 1024px, 1440px (partial — needs audit)

## 10. Migration Strategy

This design system is meant to be adopted **incrementally**, not in a single rewrite.

### Phase 1: Add tokens (non-breaking)
Add the `:root` custom properties block to the top of `style.css`. No visual changes — just defining the variables.

### Phase 2: Replace colors in new code
Any new CSS should use `var(--token-name)` instead of hardcoded hex values.

### Phase 3: Migrate existing selectors (section by section)
Replace hardcoded values with tokens one CSS section at a time:
1. Side nav (~lines 20-95)
2. Buttons (~lines 206-330)
3. Tables (~lines 335-475)
4. Cards and sections
5. Light mode overrides (simplify to just reassigning tokens)

### Phase 4: Light mode simplification
Once all selectors use tokens, light mode becomes just a `:root` override block instead of hundreds of individual selector overrides.
